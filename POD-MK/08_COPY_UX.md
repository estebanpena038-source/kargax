# 08 — Copy UX final

## Principio de copy

No vender “inspecciones”. Vender **prueba operativa de entrega**.

El usuario empresarial no quiere una inspección abstracta; quiere saber:

- qué se cargó,
- qué se entregó,
- qué se rechazó,
- quién transportó,
- dónde ocurrió,
- cuándo ocurrió,
- qué foto/nota/PIN respalda el cierre.

## Estados de error

### Ruta privada abierta en POD-MK

```txt
Esta evidencia pertenece a una ruta privada. Revísala desde Flota privada.
```

### Sin permiso

```txt
No tienes permiso para ver esta evidencia.
```

### No encontrado

```txt
No encontramos este expediente POD.
```

### Error de carga

```txt
No pudimos cargar la evidencia digital. Intenta de nuevo.
```

## CTA en Mis Ofertas

```txt
Evidencia
```

Tooltip opcional:

```txt
Ver fotos, manifiesto y cadena de custodia de esta ruta.
```

## Copy que se debe evitar

No usar:

```txt
Inspecciones
Auditoría privada
Liquidación
Comprobante de flota
Firma privada
Payroll
```

En POD-MK, evitar “firma” como requisito si no está implementada para marketplace. Sí se puede decir “POD/PIN/foto/nota/GPS”.

KargaX convierte cada ruta pública del marketplace en un expediente de evidencia digital: cargue, entrega, fotos, novedades, GPS, PIN/POD y cadena de custodia.
